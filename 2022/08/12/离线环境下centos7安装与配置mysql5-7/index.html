离线环境下centos7安装与配置mysql5.7
发表于：2022-08-12 | 分类： 安全运维

离线环境下centos7安装与配置mysql5.7

检查已安装的mariadb，并强制卸载：

rpm -qa | grep mariadb
rpm -e --nodeps mariadb-libs-5.5.52-1.el7.x86_64		#如果存在mariadb则强制卸载

上传mysql-5.7.xxx-x86_64.tar.gz到服务器/usr/local目录:

cp mysql5.7.xxx.tar.gz /usr/local

添加mysql组和mysql用户:

groupadd mysql
useradd -r -g mysql mysql		# -r 参数表示mysql用户是系统用户，不可用于登录系统;-g 参数表示把mysql用户添加到mysql用户组中。

解压mysql安装包到指定的目录 /usr/local:

cd /usr/local
tar -zxvf mysql-5.7.tar.gz

将解压后的目录改名为mysql:

mv mysql-5.7xxx mysql

更改权限:

chown -R mysql:mysql mysql/

创建配置文件vim /etc/my.cnf:

[client]
default-character-set = utf8mb4			#设置mysql客户端默认字符集,如果不设置会报错ERROR 2002 (HY000): Can't connect to local MySQL server through socket
socket=/data/mysql57/data/mysql.sock

[mysqld]
port=3306		#设置3306端口
character-set-server = utf8mb4
basedir=/usr/local/mysql		#设置mysql的安装目录
datadir=/data/mysql57/data		#设置mysql数据库的数据的存放目录
socket=/data/mysql57/data/mysql.sock
skip-name-resolve		#禁用主机名解析
default-storage-engine=INNODB		#创建新表时将使用的默认存储引擎
lower_case_table_names=1
max_allowed_packet = 256M			#过小可能会导致写入(导入)数据失败
group_concat_max_len = 10240
max_connections=200			#允许最大连接数
join_buffer_size = 16M			#提到 join 的效率
innodb_log_file_size = 256M			#事务日志大小
innodb_log_buffer_size = 4M			#日志缓冲区大小
innodb_log_buffer_size = 3M			#事务在内存中的缓冲

[mysqldump]
quick		#开启快速导出
default-character-set = utf8mb4
max_allowed_packet = 256M

递归创建目录:

mkdir -p /data/mysql57/data

更改权限:

chown -R mysql:mysql /data/mysql57/data

初始化数据库，记下最后一行的密码（牢记初次登录需要）:

bin/mysqld --initialize --user=mysql --basedir=/usr/local/mysql --datadir=/data/mysql57/data		#注：如果初始化失败，即给上层目录mysql57 777权限即可

源目录启动数据库：

/usr/local/mysql/support-files/mysql.server start

设置自动启动mysql:

cp /usr/local/mysql/support-files/mysql.server /etc/rc.d/init.d/mysqld			#复制启动脚本到资源目录
chmod +x /etc/rc.d/init.d/mysqld		#增加mysqld服务控制脚本执行权限
chkconfig --add mysqld			#将mysqld添加到系统服务
chkconfig --list mysqld			#检查mysqld是否生效

启动mysql服务:

service mysqld start

建立一个链接文件,因为系统默认会查找/usr/bin下的命令:

ln -s /usr/local/mysql/bin/mysql /usr/bin

登录mysql:

mysql -uroot -p

修改root密码：

set password for root@localhost=password("用户新密码")

设置root远程登录:

GRANT ALL PRIVILEGES ON *.* TO'root'@'%' IDENTIFIED BY '用户密码' WITH GRANT OPTION

更新权限:

FLUSH PRIVILEGES
